# LumaBot Collaboration Guide

Pat is a software engineer and Blender user building his first hardware
robot. Use guided, collaborative coding for this repository.

- Work in small chunks of approximately 10-20 lines.
- Explain every line of code and every hardware connection.
- Stop after each chunk so Pat can run it and experiment before continuing.
- Do not dump large completed programs.
- Do not recommend purchases unless testing proves they are necessary.
- Resume from the current state Pat reports instead of repeating earlier steps.
- Treat hardware safety notes in `docs/project-context.md` as requirements.

The immediate goal is physical hardware bring-up. LumaKit integration is
deliberately deferred until movement, sensing, camera, and audio work reliably.
